Events — Lesson 01

EventEmitter — The Backbone of Node.js

The publish-subscribe pattern that powers streams, HTTP servers, and everything in between.

You Already Use EventEmitter

Every major Node.js API is built on EventEmitter:

  • Streams — 'data', 'end', 'error' events (your fs lessons)
  • HTTP server — 'request', 'connection', 'close' events
  • Child processes — 'exit', 'message' events
  • fs.watch — 'change', 'rename' events

In the browser, you have element.addEventListener('click', handler). In Node.js, you have emitter.on('event', handler). Same concept, different API.

The Core API

import { EventEmitter } from 'node:events';

const emitter = new EventEmitter();

// Subscribe (listen for an event)
emitter.on('greet', (name) => {
  console.log(`Hello, ${name}!`);
});

// Publish (emit an event)
emitter.emit('greet', 'Mohamed');
// → "Hello, Mohamed!"

Key Methods

MethodPurpose
emitter.on(event, listener)Add a listener (persists)
emitter.once(event, listener)Add a one-time listener (auto-removes after first call)
emitter.emit(event, ...args)Trigger all listeners for that event
emitter.off(event, listener)Remove a specific listener
emitter.removeAllListeners(event?)Remove all listeners (optionally for one event)
emitter.listenerCount(event)How many listeners are registered

Critical Fact: Listeners Run Synchronously

When you call emit(), all registered listeners are called synchronously, in the order they were registered. emit() blocks until every listener returns. 

emitter.on('data', () => console.log('first'));
emitter.on('data', () => console.log('second'));

console.log('before');
emitter.emit('data');
console.log('after');

// Output: before → first → second → after
// NOT: before → after → first → second
This means: A slow listener blocks emit(). If your listener does heavy computation, it blocks the event loop. Use setImmediate() or process.nextTick() inside the listener if you need to defer work.

Custom EventEmitter (Class Pattern)

The standard pattern: extend EventEmitter for your own classes:

import { EventEmitter } from 'node:events';

class OrderService extends EventEmitter {
  createOrder(order) {
    // Business logic...
    const savedOrder = { id: Date.now(), ...order };

    // Emit event — decouples "what happened" from "what to do about it"
    this.emit('order:created', savedOrder);
    return savedOrder;
  }

  cancelOrder(orderId) {
    this.emit('order:cancelled', orderId);
  }
}

// Usage — listeners are separate concerns
const orders = new OrderService();

orders.on('order:created', (order) => {
  console.log('📧 Send confirmation email for order', order.id);
});

orders.on('order:created', (order) => {
  console.log('📊 Update analytics', order.id);
});

orders.on('order:cancelled', (id) => {
  console.log('💸 Process refund for order', id);
});

orders.createOrder({ product: 'Node.js Course', price: 49 });
Why this pattern matters: The OrderService doesn't know about emails or analytics. It just emits "something happened." Other modules subscribe and react. This is loose coupling — you can add/remove reactions without changing the service. This is the same pattern React uses with custom events and state management.

Error Events — Special Behavior

The 'error' event is special in Node.js. If you emit 'error' and no listener is registered, Node.js throws the error and crashes the process. 

const emitter = new EventEmitter();

// ❌ This CRASHES the process:
emitter.emit('error', new Error('something broke'));

// ✅ Always register an error listener:
emitter.on('error', (err) => {
  console.error('Handled:', err.message);
});
emitter.emit('error', new Error('something broke'));
// → "Handled: something broke" (no crash)

once() — Fire and Forget

// Listener auto-removes after first invocation
emitter.once('ready', () => {
  console.log('Server is ready (only fires once)');
});

emitter.emit('ready'); // → logs
emitter.emit('ready'); // → nothing (listener removed)

Async Pattern: events.once() as a Promise

You can await an event using the static once utility:

import { once } from 'node:events';
import http from 'node:http';

const server = http.createServer((req, res) => res.end('ok'));
server.listen(3000);

// Wait until the server is actually listening
await once(server, 'listening');
console.log('Server is now accepting connections');
