HTTP — Lesson 04

Request Body Parsing

JSON, form data, size limits — safely reading what the client sends.

Why Body Parsing Isn't Automatic

Unlike fetch().json() in the browser, Node's http module gives you a raw stream. You must:

  1. Collect chunks from the Readable stream
  2. Concatenate them into a complete string/buffer
  3. Parse based on Content-Type
  4. Validate the result

A Reusable Body Parser

async function readBody(req, { maxSize = 1_048_576 } = {}) {
  // Default max: 1MB — prevent memory attacks
  const chunks = [];
  let size = 0;

  for await (const chunk of req) {
    size += chunk.length;
    if (size > maxSize) {
      throw Object.assign(
        new Error('Request body too large'),
        { statusCode: 413 }
      );
    }
    chunks.push(chunk);
  }
  return Buffer.concat(chunks).toString();
}

async function readJSON(req) {
  const contentType = req.headers['content-type'] || '';
  if (!contentType.includes('application/json')) {
    throw Object.assign(
      new Error('Expected Content-Type: application/json'),
      { statusCode: 415 }
    );
  }

  const raw = await readBody(req);
  try {
    return JSON.parse(raw);
  } catch {
    throw Object.assign(
      new Error('Invalid JSON body'),
      { statusCode: 400 }
    );
  }
}

Using It in a Route

import http from 'node:http';

http.createServer(async (req, res) => {
  const json = (data, status = 200) => {
    res.writeHead(status, { 'Content-Type': 'application/json' });
    res.end(JSON.stringify(data));
  };

  try {
    if (req.method === 'POST' && req.url === '/users') {
      const body = await readJSON(req);

      // Validate required fields
      if (!body.name || !body.email) {
        return json({ error: 'name and email are required' }, 422);
      }

      // Create user (fake DB for now)
      const user = { id: Date.now(), ...body };
      json(user, 201);
    } else {
      json({ error: 'Not Found' }, 404);
    }
  } catch (err) {
    json({ error: err.message }, err.statusCode || 500);
  }
}).listen(3000);

Form Data (application/x-www-form-urlencoded)

HTML forms send data in this format. Parse with URLSearchParams:

async function readFormData(req) {
  const raw = await readBody(req);
  return Object.fromEntries(new URLSearchParams(raw));
}

// "name=Mohamed&email=m@test.com" → { name: 'Mohamed', email: 'm@test.com' }

Security: Size Limits

Always limit request body size. Without a limit, an attacker can send a 10GB body and crash your server with OOM (out of memory). The readBody function above enforces a 1MB default — adjust per endpoint (file uploads may need more).
