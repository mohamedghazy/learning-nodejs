Lesson 06

Directories, Paths & import.meta.dirname

Build paths that work cross-platform, navigate directories like a pro, and never get bitten by "wrong working directory" bugs.

The Path Problem

Hardcoded paths like './config.json' resolve relative to the process working directory (process.cwd()) — not relative to your source file. If someone runs your script from a different directory, paths break: 

# Your script is at /app/src/server.js
# It reads './config.json' expecting /app/src/config.json

# ✓ Works when run from the script's directory:
cd /app/src && node server.js

# ✗ Breaks when run from project root:
cd /app && node src/server.js
# Looks for /app/config.json instead of /app/src/config.json

The node:path module and import.meta.dirname solve this.

import.meta.dirname — Your Script's Directory

Available since Node 20.11.0, import.meta.dirname gives you the absolute path of the directory containing the current module. It's the ESM equivalent of __dirname in CommonJS. 

// /app/src/server.js
console.log(import.meta.dirname);
// → '/app/src'  (always, regardless of where you run the command)

console.log(import.meta.filename);
// → '/app/src/server.js'
PropertyReturnsCommonJS Equivalent
import.meta.dirname Directory of current file __dirname
import.meta.filename Full path of current file __filename
process.cwd() Where node was invoked from Same
The rule: Use import.meta.dirname when referencing files relative to your source code (configs, templates, static assets bundled with your module). Use process.cwd() when referencing files relative to the project root (user configs, data directories, workspaces). In Next.js, process.cwd() always gives you the project root where package.json lives.

The node:path Module

Never concatenate paths with string templates (`${dir}/${file}`). Use path utilities — they handle platform differences (Windows \ vs Unix /) and normalize .. and double separators.

The Functions You'll Use Daily

import { join, resolve, basename, extname, dirname, sep } from 'node:path';
FunctionWhat It DoesExample
join(...parts) Concatenate path segments, normalize separators join('src', 'lib', 'utils.js') → 'src/lib/utils.js'
resolve(...parts) Build an absolute path (resolves from right to left) resolve('src', 'config.json') → '/app/src/config.json'
basename(path) Filename from full path basename('/app/src/index.js') → 'index.js'
extname(path) File extension extname('photo.png') → '.png'
dirname(path) Parent directory dirname('/app/src/index.js') → '/app/src'
sep Platform separator '/' on macOS/Linux, '\\' on Windows

join vs resolve — When to Use Which

import { join, resolve } from 'node:path';

// join: concatenates segments, normalizes, result may be relative
join('src', 'lib', 'utils.js');
// → 'src/lib/utils.js' (relative)

join('/app', 'src', '../config.json');
// → '/app/config.json' (normalizes ..)

// resolve: always produces an absolute path
// If no absolute segment is provided, uses process.cwd() as base
resolve('src', 'config.json');
// → '/current/working/dir/src/config.json' (absolute)

resolve('/app', 'src', 'config.json');
// → '/app/src/config.json' (absolute)
Simple rule:
  • join — when you already have an absolute base path (like import.meta.dirname) and want to append segments
  • resolve — when you need to guarantee an absolute path from potentially relative input

The Full-Stack Pattern: Referencing Files

import { readFile } from 'node:fs/promises';
import { join } from 'node:path';

// ✅ File relative to this module (always correct)
const template = await readFile(
  join(import.meta.dirname, 'templates', 'email.html'),
  'utf8'
);

// ✅ File relative to project root (common in Next.js)
const config = await readFile(
  join(process.cwd(), 'config', 'settings.json'),
  'utf8'
);

// ❌ Fragile: depends on where `node` was run from
const broken = await readFile('./templates/email.html', 'utf8');

Directory Operations

Create Directories: mkdir

import { mkdir } from 'node:fs/promises';

// Create nested directories (like mkdir -p)
// No error if they already exist
await mkdir('output/reports/2026', { recursive: true });

List Directory Contents: readdir

import { readdir } from 'node:fs/promises';
import { join } from 'node:path';

// Basic: array of filenames
const names = await readdir('src');
// → ['index.js', 'utils.js', 'components']

// Better: Dirent objects (know if file or directory without extra stat call)
const entries = await readdir('src', { withFileTypes: true });

for (const entry of entries) {
  const fullPath = join('src', entry.name);
  if (entry.isFile()) {
    console.log('File:', fullPath);
  } else if (entry.isDirectory()) {
    console.log('Dir: ', fullPath);
  }
}

// Recursive: list all files in entire tree (Node 18.17+)
const allFiles = await readdir('src', { recursive: true });
// → ['index.js', 'utils.js', 'components', 'components/Button.js', ...]
Always use { withFileTypes: true } when you need to distinguish files from directories. Without it, you'd have to call stat() on each entry — which is an extra syscall per item. withFileTypes gets that info in a single operation.

Remove Files and Directories: rm

import { rm } from 'node:fs/promises';

// Remove a single file
await rm('temp.log');

// Remove a directory tree (like rm -rf)
await rm('output/old-reports', { recursive: true, force: true });
// force: true → no error if path doesn't exist
Note: rmdir with the recursive option is deprecated. Use rm with { recursive: true } instead — it's the modern replacement (available since Node 14.14).

Rename/Move: rename

import { rename } from 'node:fs/promises';

// Rename a file
await rename('draft.md', 'published.md');

// Move a file to another directory
await rename('uploads/temp.png', 'public/images/avatar.png');

Get File Info: stat

import { stat } from 'node:fs/promises';

const info = await stat('package.json');

console.log(info.size);         // file size in bytes
console.log(info.isFile());     // true
console.log(info.isDirectory());// false
console.log(info.mtime);        // last modified date
console.log(info.birthtime);    // creation date

Real-World Pattern: Walk a Directory Tree

A recursive directory walker — useful for build tools, static site generators, or finding all files matching a pattern: 

import { readdir } from 'node:fs/promises';
import { join, extname } from 'node:path';

async function findFiles(dir, extension) {
  const results = [];
  const entries = await readdir(dir, { withFileTypes: true });

  for (const entry of entries) {
    const fullPath = join(dir, entry.name);
    if (entry.isDirectory()) {
      // Recurse into subdirectories
      const subResults = await findFiles(fullPath, extension);
      results.push(...subResults);
    } else if (extname(entry.name) === extension) {
      results.push(fullPath);
    }
  }
  return results;
}

// Find all .js files in src/
const jsFiles = await findFiles('src', '.js');
console.log(jsFiles);

// Or use the built-in recursive option (Node 18.17+):
const allEntries = await readdir('src', { recursive: true, withFileTypes: true });
const jsFilesBuiltin = allEntries
  .filter(e => e.isFile() && extname(e.name) === '.js')
  .map(e => join(e.parentPath, e.name));

Real-World Pattern: Ensure Output Directory Exists

A common pattern before writing files — make sure the target directory exists: 

import { mkdir, writeFile } from 'node:fs/promises';
import { dirname, join } from 'node:path';

async function writeFileSafe(filePath, data) {
  // Create parent directories if they don't exist
  await mkdir(dirname(filePath), { recursive: true });
  await writeFile(filePath, data, 'utf8');
}

// Works even if output/reports/ doesn't exist yet
await writeFileSafe(
  join(process.cwd(), 'output', 'reports', 'summary.json'),
  JSON.stringify({ total: 42 })
);

The Legacy __dirname Story

In CommonJS (require), you had __dirname and __filename as globals. In ESM (import), they don't exist. Before Node 20.11, you had to derive them: 

// ❌ Old workaround (before Node 20.11)
import { fileURLToPath } from 'node:url';
import { dirname } from 'node:path';
const __filename = fileURLToPath(import.meta.url);
const __dirname = dirname(__filename);

// ✅ Modern (Node 20.11+) — just use it directly
console.log(import.meta.dirname);
console.log(import.meta.filename);

You'll see the old fileURLToPath pattern in many existing codebases and tutorials. Now you know it's no longer needed for Node 20.11+.
