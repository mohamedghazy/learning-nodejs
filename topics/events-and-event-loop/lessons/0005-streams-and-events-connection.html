Events — Lesson 05

How Everything Connects

Streams, HTTP servers, and the event-driven architecture that ties your Node.js knowledge together.

The Inheritance Chain

Almost every important Node.js class extends EventEmitter:

// Streams
Readable  → Stream → EventEmitter
Writable  → Stream → EventEmitter
Transform → Duplex → Stream → EventEmitter

// HTTP
http.Server         → net.Server → EventEmitter
http.IncomingMessage → Readable  → Stream → EventEmitter
http.ServerResponse  → Writable  → Stream → EventEmitter

// Other
net.Socket    → Duplex → Stream → EventEmitter
child_process → EventEmitter
fs.FSWatcher  → EventEmitter

This is why your knowledge compounds. Learn EventEmitter once, understand how every Node.js API communicates.

Streams Are EventEmitters

When you wrote pipeline(readable, writable) in your fs lessons, here's what happened under the hood:

// What pipeline() does internally (simplified):
readable.on('data', (chunk) => {
  const canContinue = writable.write(chunk);
  if (!canContinue) readable.pause(); // backpressure!
});

writable.on('drain', () => {
  readable.resume(); // buffer cleared, continue reading
});

readable.on('end', () => {
  writable.end(); // no more data
});

readable.on('error', cleanup);
writable.on('error', cleanup);

Every stream event you've used ('data', 'end', 'error', 'drain', 'close') is just EventEmitter's .on() and .emit().

HTTP Server Is an EventEmitter

import http from 'node:http';

const server = http.createServer();

// These are all EventEmitter events:
server.on('request', (req, res) => { /* handle HTTP request */ });
server.on('connection', (socket) => { /* new TCP connection */ });
server.on('listening', () => { /* server started */ });
server.on('close', () => { /* server stopped */ });
server.on('error', (err) => { /* port in use, etc. */ });

// createServer(handler) is just shorthand for:
// server.on('request', handler)

server.listen(3000);

Building a Real-Time Notification System

Combining EventEmitter with HTTP — Server-Sent Events (SSE):

import http from 'node:http';
import { EventEmitter } from 'node:events';

const notifications = new EventEmitter();

http.createServer((req, res) => {
  if (req.url === '/events') {
    // SSE: keep connection open, stream events to client
    res.writeHead(200, {
      'Content-Type': 'text/event-stream',
      'Cache-Control': 'no-cache',
      'Connection': 'keep-alive',
    });

    const onNotification = (data) => {
      res.write(`data: ${JSON.stringify(data)}\n\n`);
    };

    notifications.on('new', onNotification);
    req.on('close', () => {
      notifications.off('new', onNotification); // cleanup!
    });
  }
  else if (req.method === 'POST' && req.url === '/notify') {
    // Emit notification to all connected clients
    notifications.emit('new', { message: 'New order!', time: Date.now() });
    res.writeHead(200);
    res.end('sent');
  }
}).listen(3000);

This pattern connects: EventEmitter for pub/sub, streams (res is a Writable), HTTP server (is an EventEmitter), and cleanup on 'close' event to prevent leaks.
