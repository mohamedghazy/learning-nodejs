HTTP — Lesson 03

Routing, URL Parsing & Query Parameters

Deciding what to do based on the method and URL — the foundation every framework builds on.

What Is Routing?

Routing = matching a request's method + URL to the right handler function. Frameworks like Express automate this, but understanding the raw pattern helps you debug and build APIs correctly. 

// The core idea: method + path → handler
if (req.method === 'GET' && req.url === '/users') { /* list users */ }
if (req.method === 'POST' && req.url === '/users') { /* create user */ }
if (req.method === 'GET' && req.url === '/health') { /* health check */ }

Parsing URLs With new URL()

req.url is a raw string like /users?page=2&limit=10. The built-in URL class (same API as in the browser) parses it: 

import http from 'node:http';

http.createServer((req, res) => {
  // URL needs a base — use the request host
  const url = new URL(req.url, `http://${req.headers.host}`);

  console.log(url.pathname);            // '/users'
  console.log(url.searchParams.get('page'));  // '2'
  console.log(url.searchParams.get('limit')); // '10'

  res.end('ok');
}).listen(3000);

Dynamic Route Parameters

For URLs like /users/42 or /posts/hello-world, you need to extract the dynamic segment. Without a framework, you split the pathname: 

const url = new URL(req.url, `http://${req.headers.host}`);
const parts = url.pathname.split('/').filter(Boolean);
// '/users/42' → ['users', '42']

if (req.method === 'GET' && parts[0] === 'users' && parts[1]) {
  const userId = parts[1]; // '42'
  // fetch user by ID...
}

A Complete Router Pattern

import http from 'node:http';

http.createServer(async (req, res) => {
  const url = new URL(req.url, `http://${req.headers.host}`);
  const { pathname } = url;
  const { method } = req;

  // JSON response helper
  const json = (data, status = 200) => {
    res.writeHead(status, { 'Content-Type': 'application/json' });
    res.end(JSON.stringify(data));
  };

  // Routes
  if (method === 'GET' && pathname === '/health') {
    json({ status: 'ok', uptime: process.uptime() });
  }
  else if (method === 'GET' && pathname === '/users') {
    const page = Number(url.searchParams.get('page')) || 1;
    json({ users: [], page });
  }
  else if (method === 'GET' && pathname.startsWith('/users/')) {
    const id = pathname.split('/')[2];
    json({ id, name: `User ${id}` });
  }
  else {
    json({ error: 'Not Found' }, 404);
  }
}).listen(3000, () => console.log('http://localhost:3000'));
This is what Express does under the hood. Express adds pattern matching (/users/:id), middleware chains, and cleaner API — but it's the same method + pathname matching at the core.
