HTTP — Lesson 05

JSON API Response Patterns

Consistent response shapes, proper error responses, and the patterns that make APIs a pleasure to consume.

Consistent Response Shape

As a frontend dev, you know the pain of inconsistent APIs. As a backend dev, design your responses so clients can predict the structure: 

// Success: always has "data"
{ "data": { "id": 1, "name": "Mohamed" } }
{ "data": [{ "id": 1 }, { "id": 2 }], "meta": { "total": 42, "page": 1 } }

// Error: always has "error"
{ "error": { "message": "User not found", "code": "NOT_FOUND" } }
{ "error": { "message": "Validation failed", "code": "VALIDATION",
  "details": [{ "field": "email", "issue": "Invalid format" }] } }

Response Helpers

function sendJSON(res, data, status = 200) {
  const body = JSON.stringify(data);
  res.writeHead(status, {
    'Content-Type': 'application/json',
    'Content-Length': Buffer.byteLength(body),
  });
  res.end(body);
}

function sendSuccess(res, data, status = 200) {
  sendJSON(res, { data }, status);
}

function sendError(res, message, code, status, details) {
  const error = { message, code };
  if (details) error.details = details;
  sendJSON(res, { error }, status);
}

// Usage:
sendSuccess(res, { id: 1, name: 'Mohamed' }, 201);
sendError(res, 'User not found', 'NOT_FOUND', 404);

Centralized Error Handling

class AppError extends Error {
  constructor(message, statusCode, code) {
    super(message);
    this.statusCode = statusCode;
    this.code = code;
  }
}

http.createServer(async (req, res) => {
  try {
    // ... routing and handlers ...

    // In any handler, throw AppError:
    throw new AppError('User not found', 404, 'NOT_FOUND');

  } catch (err) {
    if (err instanceof AppError) {
      sendError(res, err.message, err.code, err.statusCode);
    } else {
      // Unexpected error — don't leak internals to client
      console.error(err);
      sendError(res, 'Internal Server Error', 'INTERNAL', 500);
    }
  }
}).listen(3000);
Never send stack traces or internal error details to the client. They reveal your file structure, dependencies, and potential attack surfaces. Log them server-side, send a generic message to the client.

Pagination Pattern

// GET /users?page=2&limit=20
sendJSON(res, {
  data: users,
  meta: {
    total: 150,
    page: 2,
    limit: 20,
    totalPages: Math.ceil(150 / 20),
  }
});

Proper Content-Length

Remember from your fs lessons — use Buffer.byteLength, not string.length:

const body = JSON.stringify({ message: 'مرحبا' }); // Arabic
res.setHeader('Content-Length', Buffer.byteLength(body)); // ✓ bytes
// NOT body.length — that counts characters, not bytes
