HTTPS, TLS & HTTP/2

How encryption works, why HTTP/2 is faster, and what you need to know for production deployment.

HTTPS = HTTP + TLS

HTTPS is not a different protocol — it's HTTP running inside a TLS (Transport Layer Security) encrypted tunnel. The HTTP messages are identical; they're just encrypted in transit.

AspectHTTPHTTPS
Port (default)80443
EncryptedNo — anyone on the network can readYes — TLS encrypts everything
CertificatesNone requiredServer must have a TLS certificate
PerformanceSlightly faster (no handshake)TLS handshake adds ~1 round trip
Required forLocal dev onlyEverything in production

How TLS Works (Simplified)

  1. Client hello — browser sends supported cipher suites
  2. Server hello — server picks a cipher, sends its certificate (public key)
  3. Client verifies — checks certificate against trusted CAs (Certificate Authorities)
  4. Key exchange — client and server agree on a shared secret (symmetric key)
  5. Encrypted communication — all HTTP data encrypted with the symmetric key

As a backend dev, you don't implement TLS yourself. You either:

  • Use a reverse proxy (Nginx, Caddy) that handles TLS — most common in production
  • Use Node's node:https module with certificate files — for direct HTTPS
  • Use a cloud provider (Vercel, AWS ALB, Cloudflare) that terminates TLS for you

HTTPS Server in Node.js

import https from 'node:https';
import { readFileSync } from 'node:fs';

const options = {
  key: readFileSync('certs/server.key'),
  cert: readFileSync('certs/server.cert'),
};

https.createServer(options, (req, res) => {
  res.writeHead(200, { 'Content-Type': 'text/plain' });
  res.end('Secure hello!\n');
}).listen(443);
In practice: Most production Node.js apps run plain HTTP behind a reverse proxy (Nginx/Caddy/cloud LB) that handles TLS termination. Your Node process listens on an internal port (3000), the proxy handles certificates and forwards requests. This is simpler and lets you rotate certificates without restarting Node.

HTTP/2 — What's Different

HTTP/2 solves HTTP/1.1's performance limitations:

FeatureHTTP/1.1HTTP/2
ConnectionsOne request per connection (or pipelining, rarely used)Multiplexing — many requests on one connection
HeadersSent as plain text every requestHPACK compression — headers are compressed and deduplicated
FormatText-basedBinary framing — faster to parse
Server PushNot possibleServer can send resources before client asks
Requires TLS?NoTechnically no, but all browsers require HTTPS for HTTP/2

Why it matters for full-stack:

  • Multiplexing eliminates the need for bundling tricks (sprite sheets, domain sharding)
  • Header compression helps APIs with large auth tokens (JWTs can be 1KB+)
  • You don't change your application code — HTTP/2 is handled at the transport layer

HTTP/2 Server in Node.js

import http2 from 'node:http2';
import { readFileSync } from 'node:fs';

const server = http2.createSecureServer({
  key: readFileSync('certs/server.key'),
  cert: readFileSync('certs/server.cert'),
});

server.on('stream', (stream, headers) => {
  // HTTP/2 uses streams, not req/res objects
  stream.respond({ ':status': 200, 'content-type': 'text/plain' });
  stream.end('HTTP/2 response');
});

server.listen(443);
In practice: You rarely use node:http2 directly. Frameworks (Express, Fastify) and reverse proxies handle HTTP/2 automatically. Your application code stays the same.

HTTP/3 (QUIC) — Brief Mention

HTTP/3 replaces TCP with QUIC (built on UDP). Benefits: faster connection setup, better on unreliable networks (mobile). Node.js has experimental support. For now, it's handled at the infrastructure level (Cloudflare, CDNs). You don't need to code for it — your HTTP handlers work the same.

What You Actually Need for Production

  1. Always use HTTPS in production — never serve over plain HTTP
  2. Get certificates from Let's Encrypt (free) or your cloud provider
  3. Use a reverse proxy (Nginx, Caddy) or cloud load balancer for TLS termination
  4. HTTP/2 is automatic if you use HTTPS + modern proxy (Caddy does HTTP/2 by default)
  5. Set HSTS header to force browsers to always use HTTPS:
res.setHeader('Strict-Transport-Security', 'max-age=31536000; includeSubDomains');
