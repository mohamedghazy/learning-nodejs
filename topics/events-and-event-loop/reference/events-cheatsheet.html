Events & Event Loop — Quick Reference

EventEmitter API

import { EventEmitter } from 'node:events';
const emitter = new EventEmitter();

emitter.on(event, listener)       // subscribe (persistent)
emitter.once(event, listener)     // subscribe (one-time)
emitter.emit(event, ...args)      // publish (synchronous!)
emitter.off(event, listener)      // unsubscribe
emitter.removeAllListeners(event) // remove all
emitter.listenerCount(event)      // count listeners
emitter.setMaxListeners(n)        // default: 10

Key Facts

  • Listeners are called synchronously in registration order
  • 'error' event with no listener → process crash
  • Async listeners: errors don't propagate to emit() — wrap in try/catch
  • MaxListeners warning (10) = likely memory leak (listeners in a loop)

Event Loop Execution Order

1. Synchronous code (call stack)
2. process.nextTick() callbacks (microtask)
3. Promise .then/.catch callbacks (microtask)
4. Timer callbacks (setTimeout, setInterval)
5. I/O callbacks (fs, network — poll phase)
6. setImmediate callbacks (check phase)
7. Close callbacks (socket.on('close'))

Defer Work (Don't Block)

setImmediate(fn)Run after I/O (check phase) — best for yielding
process.nextTick(fn)Run before any I/O (microtask) — can starve loop!
setTimeout(fn, 0)Run in next timers phase (~1ms minimum delay)
Worker ThreadsOffload CPU work > 50ms to separate OS thread

libuv Thread Pool

  • Default: 4 threads (for fs, crypto, DNS)
  • Network I/O uses OS kernel async (not thread pool)
  • Increase: process.env.UV_THREADPOOL_SIZE = '16'
