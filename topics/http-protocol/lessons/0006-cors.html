CORS — Cross-Origin Resource Sharing

The error you've fought as a frontend dev. Now you'll fix it from the server side.

What CORS Actually Is

CORS is a browser security mechanism (not a server thing, not a Node thing). The browser blocks frontend JavaScript from reading responses from a different origin unless the server explicitly allows it via headers.

Your frontendAPI serverSame origin?
http://localhost:3000http://localhost:3000/api✅ Yes (same port)
http://localhost:3000http://localhost:4000/api❌ No (different port)
https://myapp.comhttps://api.myapp.com❌ No (different subdomain)

Origin = protocol + host + port. Any difference = cross-origin = CORS applies.

The Preflight Request (OPTIONS)

For "non-simple" requests (POST with JSON, custom headers, PUT, DELETE), the browser sends an OPTIONS request first asking "is this allowed?". Your server must respond with the right headers: 

// Browser sends:  OPTIONS /api/users
// Server must respond with CORS headers + 204 (no body)

if (req.method === 'OPTIONS') {
  res.writeHead(204, {
    'Access-Control-Allow-Origin': 'http://localhost:3000',
    'Access-Control-Allow-Methods': 'GET, POST, PUT, DELETE, PATCH',
    'Access-Control-Allow-Headers': 'Content-Type, Authorization',
    'Access-Control-Max-Age': '86400', // cache preflight for 24h
  });
  res.end();
  return;
}

A Complete CORS Handler

function setCORSHeaders(req, res) {
  const allowedOrigins = [
    'http://localhost:3000',
    'https://myapp.com',
  ];
  const origin = req.headers.origin;

  if (allowedOrigins.includes(origin)) {
    res.setHeader('Access-Control-Allow-Origin', origin);
  }

  res.setHeader('Access-Control-Allow-Methods', 'GET,POST,PUT,DELETE,PATCH');
  res.setHeader('Access-Control-Allow-Headers', 'Content-Type,Authorization');
  res.setHeader('Access-Control-Allow-Credentials', 'true');
}

http.createServer((req, res) => {
  setCORSHeaders(req, res);

  // Handle preflight
  if (req.method === 'OPTIONS') {
    res.writeHead(204);
    res.end();
    return;
  }

  // ... normal request handling ...
}).listen(4000);
Never use * with credentials: true. If you set Allow-Credentials: true, you MUST specify the exact origin — * is rejected by browsers. This is because allowing credentials from anywhere would be a massive security hole.

When CORS Doesn't Apply

  • Server-to-server requests — no browser, no CORS. Your Node backend calling another API? No CORS.
  • Same origin — frontend and API on same origin. Next.js API routes are same-origin by default.
  • Simple requests — GET with no custom headers skips preflight (but still needs Allow-Origin header for reading response).
