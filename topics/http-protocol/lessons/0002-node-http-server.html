HTTP — Lesson 02

Building an HTTP Server With Node.js

Your first server in 10 lines — createServer, the request handler, and listen().

The Simplest Server

import http from 'node:http';

const server = http.createServer((req, res) => {
  res.statusCode = 200;
  res.setHeader('Content-Type', 'text/plain');
  res.end('Hello, World!\n');
});

server.listen(3000, () => {
  console.log('Server running at http://localhost:3000/');
});

Run it: node server.js. Visit http://localhost:3000. That's a server.

What's happening:

  1. http.createServer(handler) — creates a server that calls your function for every request
  2. req — an IncomingMessage (Readable stream) containing the request
  3. res — a ServerResponse (Writable stream) you write the response to
  4. server.listen(port) — starts accepting connections

The Request Object (req)

Everything the client sent is on req:

const server = http.createServer((req, res) => {
  console.log(req.method);   // 'GET', 'POST', etc.
  console.log(req.url);      // '/users?page=2'
  console.log(req.headers);  // { host: 'localhost:3000', ... }

  // Headers are ALWAYS lowercase in Node.js
  console.log(req.headers['content-type']); // 'application/json'

  res.end('ok');
});
Important: req.url is just the path + query string — NOT the full URL. http://localhost:3000/users?page=2 → req.url === '/users?page=2'.

The Response Object (res)

You control what goes back to the client:

http.createServer((req, res) => {
  // 1. Set status code
  res.statusCode = 200;

  // 2. Set headers
  res.setHeader('Content-Type', 'application/json');
  res.setHeader('X-Request-Id', 'abc-123');

  // 3. Write body and end
  const data = { message: 'hello', timestamp: Date.now() };
  res.end(JSON.stringify(data));

  // OR use writeHead for status + headers in one call:
  // res.writeHead(200, { 'Content-Type': 'application/json' });
  // res.end(JSON.stringify(data));
});

Rules:

  • Set status and headers before writing body data
  • res.end() must be called — otherwise the client hangs waiting forever
  • res.end(data) = write final chunk + close. res.write(data) + res.end() = same thing
  • After res.end(), you cannot write more data

Streaming Responses

Since res is a Writable stream, you can stream large responses: 

import http from 'node:http';
import { createReadStream } from 'node:fs';
import { pipeline } from 'node:stream/promises';

http.createServer(async (req, res) => {
  res.writeHead(200, { 'Content-Type': 'text/plain' });
  await pipeline(
    createReadStream('big-log.txt'),
    res
  );
}).listen(3000);

Your fs/streams knowledge from Lesson 4 applies directly here.

Reading the Request Body

req is a Readable stream. The body arrives in chunks — you collect them: 

http.createServer(async (req, res) => {
  if (req.method === 'POST') {
    // Collect body chunks
    const chunks = [];
    for await (const chunk of req) {
      chunks.push(chunk);
    }
    const body = Buffer.concat(chunks).toString();
    const data = JSON.parse(body);

    res.writeHead(201, { 'Content-Type': 'application/json' });
    res.end(JSON.stringify({ received: data }));
  } else {
    res.writeHead(200, { 'Content-Type': 'text/plain' });
    res.end('Send a POST request with JSON body');
  }
}).listen(3000);

Test with: curl -X POST -H "Content-Type: application/json" -d '{"name":"Mohamed"}' http://localhost:3000

Error Handling

http.createServer(async (req, res) => {
  try {
    // ... handle request ...
  } catch (err) {
    console.error(err);
    if (!res.headersSent) {
      res.writeHead(500, { 'Content-Type': 'application/json' });
      res.end(JSON.stringify({ error: 'Internal Server Error' }));
    }
  }
}).listen(3000);
Always check res.headersSent before setting status/headers in error handlers. If you already started writing the response, you can't change the status code.
